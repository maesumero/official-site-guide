# official-site-guide

本项目是一个用于归档和发布多个独立 HTML 页面的仓库。仓库内包含若干静态页面文件，这些页面各自独立，不针对特定域名或具体网站进行设计或推广。

## 项目简介

本仓库主要服务于以下目的：
- 存放和版本管理多个独立的 HTML 页面
- 提供页面归档，便于追溯历史版本
- 支持通过 GitHub Pages 或其他静态托管服务直接发布页面内容

仓库内的页面内容可能涉及说明、引导、提示或其他信息呈现形式，所有页面均为独立 HTML 文件，彼此之间无特定导航或关联。

## 目录说明

仓库根目录下包含若干 `.html` 文件，每个文件对应一个独立的页面。目录结构如下（示例）：

```
/
├── index.html
├── page-example-1.html
├── page-example-2.html
└── README.md
```

实际文件列表以仓库内容为准。如有子目录，将单独说明。

## 页面归档说明

每个 HTML 页面均按照以下原则归档：
- 页面文件名称具有描述性，便于识别内容主题
- 页面内包含完整 HTML 结构（`<!DOCTYPE html>`、`<html>`、`<head>`、`<body>` 等）
- 页面不依赖外部资源（如外部 CSS、JavaScript 库），除非在页面内显式引用并说明
- 页面内容不包含任何具体域名、网址、营销承诺或诱导点击元素

## 维护说明

- 本仓库由维护者定期更新，新增或修改页面时遵循以下原则：
  - 保持页面独立性和简洁性
  - 不引入外部依赖或跨页面引用
  - 更新时需同步更新本 README 文件（如目录结构有变化）
- 欢迎提交 Issue 或 Pull Request，但请注意：
  - 新增页面应遵循上述归档原则
  - 不添加任何与具体网站、商业推广相关的内容
  - 修改现有页面时，请说明修改原因
